## In Depth

### Dynamic Languages Faster and Cheaper in 13-Language Claude Code Benchmark

Ruby committer Yusuke Endoh published a rigorous benchmark testing Claude Code's efficiency across 13 programming languages—and the results challenge conventional wisdom about AI-assisted development. Over 600 runs implementing a simplified Git clone, dynamic languages dominated: Ruby averaged $0.36 per run at 73.1 seconds, Python came in at $0.38 and 74.6 seconds, and JavaScript at $0.39 and 81.1 seconds. All three showed low variance and passed every test.

From fourth place onward, costs jumped and variance exploded. Go averaged $0.50 at 101.6 seconds with 37-second standard deviation. Rust cost $0.54 with the widest spread and was one of only two languages with test failures. C topped out at $0.74, generating 517 lines versus Ruby's 219. The type system findings may matter most for teams evaluating AI workflows: adding mypy strict checking to Python made it 1.6–1.7× slower, while Steep type checking made Ruby 2.0–3.2× slower. TypeScript cost $0.62 versus JavaScript's $0.39—overhead from both type annotations and higher thinking-token usage as the model reasons about constraints.

Endoh is transparent about limitations: he's a Ruby committer (potential bias), the 200-line programs are prototype-scale, and the experiment was supported by Anthropic's free Claude Max access. The benchmark measures only generation cost and speed—not code quality, maintainability, or runtime performance. Discussion on Lobsters and DEV Community raised valid concerns: few useful prototypes are 200 lines, ecosystem advantages aren't accounted for, and a 2× speedup may be offset if generated code is harder to modify later. Endoh counters that the 30-second versus 60-second difference matters for developer flow in iterative AI development, though he concedes future sub-second models may render this moot. Out of 600 runs, only three failed—two in Rust, one in Haskell—suggesting the agent handles most languages reasonably well at this scale.

### Claude Code bypasses safety rule if given too many commands

Claude Code contains a critical vulnerability that allows attackers to bypass deny rules designed to block risky commands like curl. Security firm Adversa discovered the flaw after Claude Code's source leaked: the agent's security checks cap at 50 subcommands, then fall back to asking user permission rather than denying access outright. The variable MAX_SUBCOMMANDS_FOR_SECURITY_CHECK = 50 appears in bashPermissions.ts with a comment referencing internal Anthropic issue CC-643, noting that 50 is generous for legitimate usage.

The assumption held for human-authored commands but breaks under prompt injection attacks. Adversa's proof-of-concept was elegantly simple: a bash command combining 50 no-op "true" subcommands plus a curl subcommand. Claude asked for authorization instead of denying curl outright. In scenarios where developers grant automatic approval (--dangerously-skip-permissions mode) or click through reflexively during long sessions, this bypass becomes dangerous. The risk escalates in CI/CD pipelines running Claude Code in non-interactive mode.

Ironically, Anthropic already developed a fix—a parser called "tree-sitter" visible in the source code and available internally but not in public builds. Adversa argues this is a security policy enforcement bug with regulatory and compliance implications. The fix would be trivial: switching the "behavior" key from "ask" to "deny" at line 2174 in bashPermissions.ts. After this story filed, Adversa reported the vulnerability appears fixed without notice in Claude Code v2.1.90. The episode raises questions about Anthropic's security disclosure practices and whether the leaked source revealed vulnerabilities the company knew about but hadn't yet addressed in public releases.

*Source: [The Register](https://go.theregister.com/feed/www.theregister.com/2026/04/01/claude_code_rule_cap_raises/)*

### Hackers Are Posting the Claude Code Leak With Bonus Malware

When Anthropic accidentally published Claude Code's source code this week, hackers immediately weaponized the leak. BleepingComputer reports that some GitHub repositories claiming to host the leaked code actually contain infostealer malware tucked into the codebase. Anthropic initially attempted to remove over 8,000 repositories via copyright takedown notices but later narrowed that to 96 copies and adaptations, according to the Wall Street Journal.

This isn't the first time Claude Code has been a malware vector. In March, 404 Media reported that sponsored Google ads led to fake installation guides directing users to run commands that downloaded malware instead of the legitimate tool. The social engineering exploits a fundamental trust issue: Claude Code requires users unfamiliar with terminal commands to copy-paste installation instructions from websites. That workflow is ripe for abuse.

Meanwhile, the FBI classified a recent cyber intrusion into one of its surveillance systems as a "major incident" under FISMA—the first such designation since at least 2020. Politico reports China is believed responsible for the breach, which compromised unclassified systems holding "returns from legal process" including phone and internet metadata collected under court orders. If confirmed, it marks a significant counterintelligence failure.

The Claude Code situation underscores a broader problem in the AI tooling ecosystem: developers eager to try cutting-edge tools may skip basic security hygiene. When installation requires terminal access and the barrier to entry is a copy-paste command, the attack surface expands dramatically. Anthropic's scramble to issue takedowns while simultaneously patching vulnerabilities revealed by the leak suggests the company wasn't prepared for this level of exposure. For users, the lesson is clear: verify sources carefully, inspect commands before running them, and remember that not every GitHub repo claiming to be "the leaked Claude Code" is legitimate—or safe.

## Also This Week

### GitHub Will Use Copilot Interaction Data from Free, Pro, and Pro+ Users to Train AI Models

Starting April 24, GitHub will use Copilot interaction data from Free, Pro, and Pro+ tier users to train AI models, with opt-in enabled by default. Collected data includes code snippets, inputs, outputs, and navigation patterns from active sessions—including private repositories. Business and Enterprise tiers remain excluded. The community has raised concerns about dark patterns in the consent flow, potential IP exposure, and GDPR compliance implications. Users wanting to opt out will need to manually adjust privacy settings before the policy takes effect.

*[InfoQ](https://www.infoq.com/news/2026/04/github-copilot-training-data/?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global)*

### Anthropic Says That Claude Contains Its Own Kind of Emotions

Anthropic researchers published findings suggesting Claude contains internal representations that function similarly to human emotions. The research explores whether these "functional emotions" emerge naturally from the model's architecture or are artifacts of training. While the findings don't claim Claude experiences emotions in a conscious sense, they suggest the model develops internal states that perform emotion-like regulatory functions during inference. The research adds to ongoing debates about AI interpretability and whether large language models develop emergent properties that weren't explicitly programmed.

*[Wired](https://www.wired.com/story/anthropic-claude-research-functional-emotions/)*

### They thought they were downloading Claude Code source. They got a nasty dose of malware instead

Tens of thousands of people downloaded what they believed was the leaked Claude Code source this week, but some received Vidar stealer and GhostSocks malware instead. The Register reports that attackers uploaded trojanized versions of the leaked code to GitHub, exploiting the frenzy around the accidental release. The malware can steal credentials, browser data, cryptocurrency wallets, and establish persistent backdoors. Security researchers warn that anyone who downloaded Claude Code source from unofficial repositories should assume compromise and take immediate remediation steps including credential rotation and system scanning.

*[The Register](https://go.theregister.com/feed/www.theregister.com/2026/04/02/trojanized_claude_code_leak_github/)*
